5.6.4  Parallel Programming

The difference between concurrent programming and parallel programming is:

  • In concurrent programming multiple application-program threads appear to run "at the same time" but actually run on a single core via pre-emptive timeslicing.
  • In parallel programming application-program threads run simultaneously on multiple cores; on a six-core machine a speedup of six is possible.

Low-level Mythryl support for parallel programming based based on the industry-standard posix-threads API is defined in src/lib/std/src/hostthread.api and documented in src/A.HOSTTHREAD-SUPPORT.OVERVIEW but not yet sufficiently debugged for use as of 2012-01-22.
